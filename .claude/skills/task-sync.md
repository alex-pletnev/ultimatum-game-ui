# task-sync

Аудит трекера: находит расхождения между `docs/tasks/*.md`, `docs/tasks/INDEX.md` и текущим кодом. **Ничего не правит без явного согласия пользователя.**

## Шаги

1. **Собрать реальный список задач.**
   - `Glob` `docs/tasks/T-*.md` — все файлы задач.
   - Из каждого прочитать frontmatter (`id`, `title`, `status`, `priority`, `updated`, `related_code`, `related_docs`).

2. **Прочитать INDEX.md.** Распарсить таблицы «Открытые» и «Закрытые».

3. **Сверка A: INDEX ↔ файлы.**
   - Задачи есть в файлах, но нет в INDEX — «отсутствует в индексе».
   - Задачи есть в INDEX, но нет файлов — «висячая запись».
   - Совпадение по ID, но `status`/`title`/`updated` расходится — «расхождение полей».

4. **Сверка B: related_code существует.**
   - Для каждой открытой задачи прогнать `related_code` через `Read`/`Glob`. Отсутствующие пути — «мёртвая ссылка».

5. **Сверка C: related_docs существует.**
   - То же самое для `related_docs`.

6. **Сверка D: подозрительно старые.**
   - Открытые задачи с `updated` старше 30 дней от сегодня — вывести отдельным списком «застоявшиеся» (без действия, просто пометка).

7. **Отчёт пользователю** — компактная таблица по каждой из сверок, например:

   ```
   Расхождения:
   - T-005: в INDEX status=pending, в файле in_progress
   - T-012: файл есть, в INDEX нет
   Мёртвые ссылки:
   - T-002 → path/to/OldFile.ext (удалён?)
   Застоявшиеся (>30 дней):
   - T-001, T-003
   ```

8. **Спросить пользователя, что чинить.** Не начинать правки без явного согласия. Возможные действия:
   - Обновить INDEX под файлы.
   - Обновить файлы под INDEX.
   - Обновить `related_code` (найти новое расположение через `Grep` по имени класса).
   - Пометить `blocked` или `cancelled`.

## Ограничения

- Никаких изменений без подтверждения (даже «очевидных»).
- Не создавать новые задачи. Это делает `/task-add`.
- Не закрывать задачи. Это делает `/task-done`.
