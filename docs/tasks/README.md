# Task-трекер

Простой файловый трекер задач. Одна задача = один MD-файл. Индекс — [INDEX.md](INDEX.md).

## Формат имени файла

```
T-XXX-kebab-case-title.md
```

- `XXX` — номер задачи (000-padded).
- Заголовок — kebab-case, латиница.

## Структура таск-файла

```markdown
---
id: T-XXX
title: Полное название
status: pending | in_progress | blocked | done | cancelled
priority: low | medium | high
created: YYYY-MM-DD
updated: YYYY-MM-DD
related_code:
  - path/to/file
related_docs:
  - docs/XX-name.md
tags: [frontend, api, ui, tech-debt, meta]
---

## Контекст

Зачем это нужно. Ссылка на источник.

## Acceptance criteria

- [ ] Условие 1
- [ ] Условие 2

## План

1. Шаг 1
2. Шаг 2

## Лог

- `YYYY-MM-DD`: событие.
```

## Статусы

| Статус | Когда |
|--------|-------|
| `pending` | Заведена, работа не начата |
| `in_progress` | В работе |
| `blocked` | Нужно решение / внешнее событие |
| `done` | Acceptance выполнены, docs синхронизированы |
| `cancelled` | Отменена; причина в логе |

## Правила обновления

- `updated` меняется при каждой правке.
- Любое существенное изменение — в `## Лог`.
- Перевод в `done` — только через `/task-done`.
- Отмена — `status: cancelled`, файл не удалять.

## Slash-команды

| Команда | Что делает | Git |
|---------|-----------|-----|
| `/task-add "название"` | Создаёт `T-XXX-*.md`, обновляет INDEX.md | commit (без push) |
| `/task-done T-XXX` | Меняет `status: done`, лог, INDEX.md | тесты → commit → push |
| `/task-sync` | Проверяет консистентность | нет |
| `/docs-sync` | Актуализирует `docs/*.md` | commit → push (если правки) |

Полные правила git-flow — `CLAUDE.md` → раздел «Git-автоматизация».

## Специальные теги

| Тег | Смысл |
|-----|-------|
| `tech-debt` | Проблема кода, замеченная агентом по ходу другой работы |
| `meta` | Улучшение инструментария: `CLAUDE.md`, `.claude/skills/`, формат таск-файла, структура `docs/` |

## Auto-mode: заведение задач агентом

Агент имеет право сам заводить задачи при выполнении **фильтра из трёх условий**:

1. Проблема реальная (сломанное поведение, скрытый риск, дублирование, нарушение инварианта) — не косметика.
2. Формулируется `## Контекст` в 2-3 строки и минимум один пункт `## Acceptance criteria`.
3. Проблема не относится к текущей задаче и растворится без записи.

Если хоть одно не выполнено — задача не заводится; агент упоминает наблюдение одной строкой.

Обязательные атрибуты авто-заведённой задачи:
- Тег `tech-debt` или `meta`.
- Приоритет `low` по умолчанию; `medium` для опасных; `high` — по согласованию.
- `## Лог`: `- YYYY-MM-DD: заведена автоматически в ходе работы над T-YYY.`

Правило прозрачности: агент отдельной строкой в конце ответа — «Попутно завёл T-XXX (…): краткое описание. Отменить — скажи».

Полные правила — в `CLAUDE.md` → «Проактивное заведение задач».

## Заметки для агента

- Задача рождается из явного запроса, из пункта `docs/11-known-gaps.md` или из Auto-mode.
- Перед созданием — проверить дубликаты (`grep` по INDEX и файлам).
- `related_code` / `related_docs` — обязательные поля.
